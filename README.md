# LLM Helper – MATLAB Script & Live Script Export (Flat ZIP)
This readme was autogenerated by Co-pilot 365
## TL;DR

*   Converts `.m` / `.mlx` → `.txt`
*   Flattens entire project

*   ZIP is **for transport only**
*   **Extract ZIP before uploading to Copilot**
*   Upload extracted `.txt` files + CSV together

## Overview

**LLM Helper – Flat ZIP** is a MATLAB utility designed to prepare **MATLAB / Simulink projects** for use with **Microsoft Copilot 365** and other Large Language Models (LLMs).

Copilot **cannot natively upload `.m` or `.mlx` files**, and it also **cannot accept ZIP files directly**.  
This script solves both problems by:

*   Converting `.m` and `.mlx` files into **plain `.txt`**
*   Flattening all files into a **single-level structure**
*   Providing a **CSV mapping file** to preserve original paths
*   Enabling **easy bulk upload** to Copilot after ZIP extraction

Original project files are **never modified**.

***

## Key Design Goals

*   ✅ Make MATLAB code ingestible by Copilot / LLMs
*   ✅ Preserve system-level context without folders
*   ✅ Enable one-step bulk upload workflows

***

## What the Script Does

*   Recursively scans a MATLAB / Simulink project folder
*   Finds:
    *   Native `.m` scripts and functions
    *   `.mlx` Live Scripts
*   Converts everything to `.txt`
*   **Removes all directory structure** (flat layout)
*   Resolves filename collisions automatically
*   Generates a CSV log mapping files back to their original locations
*   Packages everything into a single ZIP archive

***

## Output Files

After execution, the following files are created in the **target directory**:

### `Scripts_Archive_Flat.zip`

Contains:

*   All converted `.txt` files
*   **No subfolders** (fully flat structure)
*   The CSV log file

### `Script_Conversion_Log.csv`

Maps flat files back to their original project structure.

**Columns:**

*   `OriginalName`
*   `OriginalLocation`
*   `NewFlatName`
*   `Type` (Native `.m` → `.txt` or Converted `.mlx` → `.txt`)

This CSV is critical for restoring architectural context inside Copilot.

## Usage

Run on the current directory:

```matlab
LLM_helper_flat_zip
```

Run on a specific project folder:

```matlab
LLM_helper_flat_zip('C:\MyMatlabProject')
```

## ✅ Correct Copilot / LLM Upload Workflow (IMPORTANT)

Copilot **does NOT support uploading ZIP files directly**.

### Correct workflow:

1.  Run the script
2.  Locate `Scripts_Archive_Flat.zip`
3.  **Extract the ZIP** to a temporary folder
4.  Upload **all extracted contents** to Copilot:
    *   All `.txt` files
    *   `Script_Conversion_Log.csv`

📌 Upload **everything together** so Copilot can reason across files using the CSV for context.

## Why a Flat Structure?

LLMs:

*   Do **not reliably reason over folder hierarchies**
*   Perform better when **all files are visible at once**

The flat structure maximizes visibility.  
The CSV restores hierarchy **only when needed**.
